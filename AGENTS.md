# AGENTS.md

Personal website built with Astro v6 (fully static, no SSR adapter, no UI framework).

## Package manager

Use **npm** only. Lockfile is `package-lock.json` (lockfileVersion 2, requires npm v7+). Node v22 (see `.nvmrc`); Astro v6 requires `>=22.12.0`.

## Commands

```sh
npm run dev       # dev server → localhost:3000
npm run build     # production build → ./dist/
npm run preview   # serve ./dist/ locally
npm run astro -- check   # type-check (only type-check mechanism — no separate tsc script)
```

No lint, format, or test scripts exist.

## Type-checking

`tsconfig.json` extends `astro/tsconfigs/strict` — full strict mode. Run `npm run astro -- check` (not `tsc` directly).

## Source layout

```
src/
  env.d.ts          # Astro client type refs — do not edit manually
  layouts/
    Layout.astro    # Single base layout, accepts `title: string` prop
  pages/
    index.astro     # Home page
```

No `components/`, `content/`, `public/`, or `styles/` directories yet.

## Styling

Plain CSS only — no Tailwind, no CSS modules, no Sass. Global styles use `<style is:global>` in `Layout.astro`; page-level styles use Astro's default scoping. Theme colors are CSS custom properties (`--bg`, `--fg`).

## Icons

`astro-icon` integration with the full Lucide icon set. Import and use as:

```astro
import { Icon } from "astro-icon/components";
<Icon name="lucide:tent-tree" size="32" />
```

`.astro/icon.d.ts` is auto-generated — do not edit it manually.

## Generated files

`.astro/` is auto-generated by dev/build. Never edit files inside it.

## Gotchas

- **Favicon** is an inline SVG data URI in `Layout.astro`, not a file in `public/`.
- `Astro.site` is `undefined` — no `site` is configured in `astro.config.mjs`; avoid code that depends on absolute URL generation.
- `"type": "module"` in `package.json` — ESM only, `require()` will not work.
- No CI workflows, no pre-commit hooks, no deployment config files present.
